# GitHub Issue Mapping

GitHub issues are the primary source of truth for feature development in any repo using this skill. Sprints/cycles are organized by milestone. A repo-specific overlay skill may narrow or relabel these rules — this is the generic default.

## Rules
1. Every PR must link to an issue.
2. Issues must have a `type` label (bug, feature, tech-debt).
3. Milestones correspond to semantic versioning releases, or the repo's own release cadence.
