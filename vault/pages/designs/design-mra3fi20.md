# Sovereign Ledger: Visual Identity Spec

### High-Level Design Reasoning
Our visual strategy translates the ethos of sovereign, file-native computing into a tactile, editorial layout reminiscent of physical state archives and archival municipal ledgers. We pair the organic warmth of physical storage media with structural grid lines that reference relational data schemas. 

### 1. Real-World Visual References
- **Municipal Land Registry Ledgers (c. 1890s):** Heavy bound volumes containing handwritten technical descriptions alongside rigid mechanical grids.
- **The Basel University Library Archives:** Cast-in-place raw concrete storage monoliths safeguarding delicate paper assets.
- **Card Catalog Index Cabinets:** Repetitive brass-plated card slots holding dense metadata on small paper slips.

### 2. Typographic Juxtaposition
We pair **Cormorant Garamond** (an elegant, hyper-traditional serif with razor-thin serifs) with **PT Mono** (a utilitarian, typewriter-esque monospace). This pairing represents the intersection of humanistic curation and immutable, mechanical execution.

### 3. Palette
- **Paper Ground:** `#F5F4F0` (light warm organic texture)
- **Deep Navy Ink:** `#0F1E2C` (dense, heavy typographical foundation)
- **Muted Slate:** `#6A7B83` (mid-tone architectural framing color)
- **Oxidized Iron:** `#2C3A41` (dense metadata/borders)
- **Sovereign Accent:** `#B3543D` (terracotta-rust industrial marking ink)

### 4. Signature Structural Device
The **Asymmetric Ledger Spire**: A vertical data-rail layout where a heavy, 2-pixel navy rule splits the page asymmetrically, displaying technical index data on the left in a strict grid, and flowing editorial thesis blocks to the right.

### 5. Architectural Critique
This site rejects common portfolio trends by entirely omitting rounded corners, cards, or smooth neon gradients. Instead, its brutalist, ink-and-paper layout mirrors the structural weight of the sovereign architectures built by Greg.
