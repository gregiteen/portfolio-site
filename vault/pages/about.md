## Who I am

I'm a software engineer who thinks most modern software holds user data
hostage. My work pushes the other direction: **sovereign systems** — plain
files, open specs, and runtimes you can inspect, fork, and carry with you.

## How I work

- **Files over databases.** A vault of Markdown with semantic frontmatter can
  be a CMS, a memory store, and an API surface all at once.
- **Contracts over vibes.** AI-generated mutations flow through deterministic,
  validated operation envelopes — never raw writes.
- **Ship the spec.** If a system matters, publish the standard so others can
  implement it without you.

## Toolbox

TypeScript / Node.js, Python, Postgres, Supabase, Vercel, Express,
local embedding models, and far too many Markdown files.
